# 듀랑고 미니 — 분산 존 서버 프로토타입

탑다운 2D 멀티플레이어 생존 게임의 미니 버전. **하나의 월드를 여러 서버가 지역별로 나눠서 시뮬레이션**하는 구조를 실제로 구현한 프로토타입입니다.

월드는 서→동으로 **중국 평원 · 한반도 산악 · 일본 삼림** 3개 존으로 구성되어 있고, 각 존은 독립된 Node.js 프로세스가 권위적으로 시뮬레이션합니다. 플레이어가 동/서 경계를 넘으면 클라이언트의 WebSocket 연결이 자동으로 다음 존 서버로 전환됩니다(핸드오프).

실제 분산 배포에서는 각 존 서버를 다른 국가의 데이터센터에 두면 됩니다. 프로토타입에서는 같은 머신의 다른 포트(3001, 3002, 3003)로 시뮬레이션합니다.

## 실행 방법

```bash
npm install
npm start
```

`npm start`는 디스패처(3000)와 3개의 존 서버(3001~3003)를 한꺼번에 띄웁니다. 그다음 브라우저에서:

```
http://localhost:3000
```

이름을 입력하고 시작 존을 골라서 입장하면 됩니다. 멀티플레이를 확인하려면 같은 URL을 다른 브라우저 탭이나 시크릿 창에서 열면 됩니다.

## 조작

- **WASD / 화살표키** — 월드 방향 이동 (W=북, D=동, S=남, A=서). 아이소 시점이라 D 누르면 화면상 "오른쪽-아래" 대각선으로 움직임 — 그게 월드의 동쪽
- **옆 나라 가는 법** — 한국→일본은 그냥 **D** 길게. 화면 가장자리 노란 화살표가 인접국 방향을 가리킴. HUD의 좌표(월드/로컬)로 진행 상황 확인 가능
- **E** — 채집 (가까운 나무/돌)
- **C** — 토지 점유 (나무 5 + 돌 5)
- **T** — 거래 (나무 1 → 돌 1)
- **Y** — 거래 (돌 1 → 나무 1)
- **존 경계로 이동** — 자동 핸드오프 (인접 존을 미리 observer로 구독해서 끊김 없이)

## 구조

```
durango-mini/
├── server/
│   ├── zone-config.js   # 존 토폴로지 (공통 설정)
│   ├── zone.js          # 존 서버 (한 지역 시뮬레이션 권위)
│   └── dispatcher.js    # 정적 파일 + 존 라우팅
└── public/
    ├── index.html
    ├── style.css
    └── client.js        # Canvas 렌더링 + 입력 + WS
```

## 다음 단계로 확장하기 좋은 것들

- **건축 시스템** — 클레임한 토지 안에 벽·문·창고 짓기
- **자원 가공** — 도끼, 곡괭이 등 도구 크래프팅
- **NPC와 몬스터** — 빈 땅의 야생성 강화
- **글로벌 거래소** — 디스패처에 시장 DB를 두고 존 간 비동기 거래
- **데이터 영속화** — 현재는 메모리 저장, Redis나 Postgres 추가
- **GeoIP 라우팅** — 디스패처가 클라이언트 IP로 가장 가까운 존을 추천
- **실제 다국가 배포** — 각 존 서버를 다른 클라우드 리전에 배포

## 주요 설계 선택

**왜 클라이언트가 직접 존 서버에 붙는가** — 디스패처가 메시지를 중계하면 디스패처가 병목이 됩니다. 클라이언트가 직접 존 서버에 붙고, 디스패처는 라우팅 정보만 알려주는 게 분산 구조의 핵심.

**왜 권위는 존 서버에 있는가** — 같은 지역 안의 플레이어들끼리 상호작용할 때 단일 권위 서버를 거치는 게 일관성 유지에 가장 단순.

**왜 핸드오프는 WS 재연결로 처리하는가** — 존 간 직접 통신(P2P 서버) 대신 클라이언트를 통한 리레이가 프로토타입에선 훨씬 간단. 본격 운영 시에는 존 간 백엔드 메시지 버스(Redis pub/sub, NATS) 추가 권장.

**왜 클라이언트 예측이 있는가** — 서버 tick이 10Hz라서 즉시 반응이 안 보임. 입력 즉시 클라가 예측하고, 서버 권위 좌표와 어긋나면 보정.
